FFTLIB_FFT_highRadixDecompositions_ixX_cxX_oxX
Fast Fourier Transform (FFT) kernels

Introduction

Kernel for computing DFT by using a high-radix variant of the FFT alogrithm.

FFTLIB_FFT_highRadixDecompositions_ixX_cxX_oxX

     - The size of data for each channel in the batch is assumed to be
     greater than or equal to (MMA vector width)*2, and is assumed to
     be a power of 2.
     - The kernel requires application to provide a pre-generated
     computational matrices along with the input data.<br>
     - The algorithm works by decomposing DFT computation to smaller
     size DFT's stage after stage. In all but the first stage, the
     decomposition factor is (MMA vector width)/2. This decomposition
     factor is chosen so that each row of the decomposition matrix would
     fit one full vector length. The first stage would also use a
     decomposition factor of (MMA vector width)/2 if the data size

(fftSize) is a power of (MMA vector width)/2. In case the data size is not a power of (MMA vector width)/2, the decomposition factor for the first stage is less than (MMA vector width)/2.

#define FFTLIB_FFT_HIGHRADIXDECOMPOSITIONS_IXX_CXX_OXX_NUMSHIFTS   5

Macro to define the number of bit-shifts that are to be provided for use in the internal computation.

The number of bit-shifts depends on the number of stages of computation - that is number of decomposition stages. Assuming a maximum data size of 32768 in case of 32-bit data type, maximum number of computation stages would be 5. (log(32768) base 8)

This is a utility function that generates the computational matrices (DFT and twiddle factor matrices) into the provided buffer. Part of the twiddle factors and DFT matrices are generated into the FFTLIB_FFT_twData buffer that must be mapped to L1D memory for optimal performance. Note that this buffer is different from the buffer pointed to by the pW argument that is passed to this function.

Returns
Status value indicating success or failure. Refer to FFTLIB_STATUS.
Remarks

Assumptions:
  • None
Performance Considerations:
For best performance,
  • the input and output data buffers are expected to be in L2 memory
  • the buffer pointers are assumed to be 64-byte aligned
Remarks
Before calling this function, application is expected to call FFTLIB_FFT_highRadixDecompositions_ixX_cxX_oxX_init and FFTLIB_FFT_highRadixDecompositions_ixX_cxX_oxX_exec_checkParams functions. This ensures resource configuration and error checks are done only once for several invocations of this function.
