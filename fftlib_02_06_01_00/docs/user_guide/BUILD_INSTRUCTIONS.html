FFTLIB Build Instructions


Build instructions for FFTLIB

Setup
  • Run the executables on EVM using CCS (Code composer studio)
  • Setup needed the compiler for CGT7x
    • export CGT7X_ROOT=${FFTLIB_CGT7X_COMPILER_DIR}
  • Setup for HOST EMULATION
    • export C7X_HOST_EMULATION_PATH=${CGT7X_ROOT}/host_emulation
  • Path setup
    • export PATH=${CGT7X_ROOT}/bin:${PATH}
Build
  • Compile the code from the FFTLIB_xx_xx_xx_xx folder
  • Compile the libraries for target
    • make -j16 fftlib fftlib_cn common test TARGET_CPU={C7100, C7120, RESERVED} TARGET_BUILD=release CHECKPARAMS=1 SOC={J7ES, J7AEP, RESERVED} TEST_ENV=EVM
      • libraries generated will be /out/C7100/release or /out/C7120/release or /out/RESERVED/release
      • make -j16 fftlib fftlib_cn common test TARGET_CPU={C7100, C7120, RESERVED} TARGET_BUILD=debug CHECKPARAMS=1 SOC={J7ES, J7AEP, RESERVED} TEST_ENV=EVM
      • libraries generated will be /out/C7100/debug or /out/C7120/debug or /out/RESERVED/debug
      • C7100: common_C7100.lib, fftlib_cn_C7100.lib, fftlib_C7100.lib, test_C7100.lib
      • C7120: common_C7120.lib, fftlib_cn_C7120.lib, fftlib_C7120.lib, test_C7120.lib
      • RESERVED: common_RESERVED.lib, fftlib_cn_RESERVED.lib, fftlib_RESERVED.lib, test_RESERVED.lib
  • Compile the libraries for host
    • make -j16 fftlib fftlib_cn common TARGET_CPU=x86_64 TARGET_SCPU={C7100, C7120, RESERVED} TARGET_PLATFORM=PC TARGET_BUILD=release CHECKPARAMS=1
    • libraries build libfftlib_x86_64.a, libfftlib_cn_x86_64.a, libcommon_x86_64.a, libtest_x86_64.a
  • Compiling files to run on EVM with release or debug build in source package
    • make -j8 {module folder, example fft_c7x}/{module name, example FFTLIB_fft1dBatched_i16sc_c16sc_o16sc} TEST_ENV=EVM EXTENDED_TEST_CASE={39} TARGET_CPU={C7100, C7120, RESERVED} TARGET_BUILD=release CHECKPARAMS=1 SOC={J7ES, J7AEP, RESERVED}
    • make -j8 {module folder, example fft_c7x}/{module name, example FFTLIB_fft1dBatched_i16sc_c16sc_o16sc} TEST_ENV=EVM EXTENDED_TEST_CASE={39} TARGET_CPU=RESERVED TARGET_BUILD=debug CHECKPARAMS=1 SOC=RESERVED
  • Compile the individual test for host for a given component example FFTLIB_fft1dBatched_i16sc_c16sc_o16sc
    • make -j8 fft_c7x/FFTLIB_fft1dBatched_i16sc_c16sc_o16sc EXTENDED_TEST_CASE=39 TARGET_CPU=x86_64 TARGET_SCPU=C7100 TARGET_PLATFORM=PC TARGET_BUILD=release CHECKPARAMS=1
  • Compile the individual test on EVM with binary package
    • make scrub; make -j12 TARGET_CPU={C7100, C7120, RESERVED} LINK_LIBS=1 {module folder, example fft_c7x}/{component name, example FFTLIB_fft1dBatched_i16sc_c16sc_o16sc} TEST_CASE=39 TEST_ENV=EVM CHECKPARAMS=1 SOC={J7ES, J7AEP, RESERVED}
  • Compile the individual test on host emulation with binary package
    • make -j12 TARGET_SCPU=C7100 TARGET_CPU=x86_64 TARGET_PLATFORM=PC LINK_LIBS=1 {module folder, example fft_c7x}/{component name, example FFTLIB_fft1dBatched_i16sc_c16sc_o16sc} TEST_CASE=39 CHECKPARAMS=1 SOC=J7ES
Run
  • Execute a test case on host
    • ./out/x86_64/release/fft_c7x/FFTLIB_fft1dBatched_i16sc_c16sc_o16sc_x86_64
  • Execute a test case on target
    • Executing the test case on EVM is done by loading binary generated via JTAG using CCS (Code Composer Studio) out/C7100/release/fft_c7x/FFTLIB_fft1dBatched_i16sc_c16sc_o16sc_C7100.out
