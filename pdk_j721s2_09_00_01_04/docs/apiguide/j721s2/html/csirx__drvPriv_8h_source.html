72 /*
73  * State transitions are explained below
74  *
75  * IDLE - initial or not open state
76  * |
77  * CREATED - state after calling Fvid2_create
78  * |
79  * DO_START - temporary state when Fvid2_start() is called
80  * | this means driver has requested Capture Module to start the instance
81  * |
82  * RUNNING - this is the state after Fvid2_start() completes, this means
83  * | Capture Module started capture and now it is running, it remains in this
84  * | state
85  * | until Fvid2_stop() is called
86  * |
87  * DO_STOP - temporary state when Fvid2_stop is called()
88  * | this means driver has request Capture Module to stop the instance
89  * |
90  * STOPPED - this is the state after Fvid2_stop() completes, this means
91  * | Capture Module has stopped this instance
92  * |
93  * IDLE - this is the state after Fvid2_delete() completes
94  */
